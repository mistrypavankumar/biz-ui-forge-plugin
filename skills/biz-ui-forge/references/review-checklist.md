# Review Checklist

Before finalizing, verify:

1. The chosen mode matches the request.
2. The code preserves business logic.
3. The visual hierarchy matches the mockup or intent.
4. Child-owned zones were traced and updated when needed.
5. Existing reusable components were preferred over rebuilding from scratch.
6. Theme tokens are used instead of hardcoded visual values.
7. Loading, empty, error, hover, focus, and disabled states are covered.
8. The final result feels like senior-level product work, not default scaffolding.
